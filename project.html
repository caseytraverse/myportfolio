overview

An existing client had come to the agency with an idea for a new project. They had the core of an idea, but there was still a lot of unanswered questions about the specific direction for the project. With the sector still being young, the client felt there was room for another player in the cloud storage and organizational market. The client wants to enhance the user experience with collaborative and social sharing tools. The client needed further user research in this market and a brand identity. It was my pleasure to help with the design and brand development.

the problem

Currently, there is not a cloud storage service that does it all. It can be frustrating switching between various apps to perform different tasks. Users are looking for an all-in-one collaborative cloud storage platform that is secure and easy to use.

the solution

alto provides an all-in-one solution to users looking for a secure, user friendly one-stop cloud storage app. With alto, users can upload, store, create, collaborate, share and more. It’s highly accessible with both free & low-priced plans that anyone can enjoy. View prototype >

design process

Competitive Analysis

Firstly, I needed to gather information about the existing market. I began by completing a competitive analysis of three well established existing companies. After thorough research and understanding of the competitive landscape I was able to determine the best way for a competitor to enter the marketplace.

Competitive Analysis marketplace conclusion.

The cloud storage market has been a rapidly growing market for over a decade. I focused my analysis on three of the biggest companies in this sector.

Google's Drive. Apple's iCloud. Pinterest.

While each three companies offer different things to the market, none of them are the one-stop shops that users are searching for. Google Drive allows you to store, create, and share content. Apple’s iCloud is primarily a store and share app. Pinterest is a visual sharing and organizational app. While there are many cloud storage services on the market, there’s still room for growth, like alto, to enter the market and make some waves. I then created the user flows for the three companies Apple’s iCloud, Pinterest and Google Drive. This helped me to identify the priorities of the competitors and what they're directing users towards.

Pinterest's onboarding user flow.

User Survey's

To better acquaint myself with the users wants and needs I conducted a user survey. I distributed the survey across various social media platforms (Facebook, Reddit, Twitter) to gather enough responses to make user personas and user stories. In the survey, I focused on the following aspects:
  • Which features they value the most
  • Which cloud storage platform the users use
  • What are some features that could enhance their experience
  • What additional collaborative features they would like to see offered
  • If they use collaborative features on their cloud storage service

User surveys were conducted.

User Stories

Using the data from both my competitive analysis and user surveys, I was able to create informed user stories. I compiled a list of tasks that new and returning users need to complete when they would use the site. I ordered the tasks from high, medium to low priority.

User stories were compiled.

User Flows

In order to visualize my user stories I followed by creating user flows. User flows are the visualized conception of the steps, or path the user would take navigating specific tasks on a website or app. I converted all of my high priority user stories as well as some medium priority user stories into user flows. This helped assist me to visualize the processes that are necessary to be completed for a minimum viable product.

Organizing files - user flow.

Usability testing

Usability testing is very valuable to the process as it helps you understand what may or may not be working in your design before it gets too far. This feedback helped me conceptualize my visual design for my high fidelity prototype.

Branding - Research & Sketching

It was necessary that the branding message conveys a sense of safety, security and dependability. This was seen as an important aspect for a cloud storage brand to possess according to research data. I used this as my inspiration as I began to sketch out various logo designs. I decided to name the platform alto, after altocumulus clouds. alto is short, modern and simple.

Initial logo sketches.

Preference testing

Before testing my high fidelity mockup, I conducted preference tests on three different iterations of a few design elements. I tested user preferences on the dashboard layout, sidebar design and readability.

Gathering preference testing data.

High fidelity usability testing

After additional design revisions based off of the preference testing data, I was able to move forward with another round of usability testing. I tested my high fidelity prototype with three seperate users.

Usability testing on high fidelity prototypes.

What worked:

  • User experience. Participants felt that the usability of the site was both user friendly and easy to navigate.
  • Brand recognition. Participants were able to identify from the landing page that alto is a cloud storage organizational/social sharing platform.

What needed improvement:

  • Contrast. Participants felt that there was not enough color contrast and they experienced some readability issues.

Reflection

This project was a challenge for me, however, the experience was incredibly valuable. Going into this project I was a bit apprehensive as the client brief was extremely vague. My biggest concern was in regards to materializing an idea with so little direction. What I soon learned surprised me the most about this experience. Even without a clear vision for a project, you can still find a purposeful direction utilizing the resources of user research. Throughout the design process, I was highly impacted by how important user testing is. Of course I would know how to navigate and use a website I designed. But how would a new user react to the flow of the website? That was the most useful and interesting information to collect. What seemed to work was both brand recognition and the overall usability of the site. What I needed to work the most on was the visual hierarchy and the contrast/readability of the site. If I had more time I would have expanded my usability testing to more users from around the globe. For my usability testing I chose participants readily available to me, which narrowed my testing to a smaller geographical location. Ultimately, I think this project showcases my abilities to accualized an ambiguous client brief into a fully conceptualized finished design.
